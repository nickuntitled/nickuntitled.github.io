ใช้ไม่ถึงเดือนก็ส่ง Zenfone 5 เข้าศูนย์แล้ว กับแนะนำการทำ Factory Reset

Zenfone 5 เป็นโทรศัพท์ Android ในแบรนด์อย่าง ASUS ที่ทำราคาได้ถูกเมื่อเทียบกับรุ่นอื่น มีสองซิมและสเปคดี เสียอย่างเดียว (เท่าที่ใช้) คือแบตหมดเร็วไปหน่อย

เราก็เลยซื้อมาใช้ หลังจากที่ซื้อมาใช้อาการก็ปกติ ไม่มีปัญหาอะไรที่รบกวนใจเรา

พอใช้ไปไม่ถึงเดือนก็เกิดอาการสองอย่างขึ้นมาเป็นเวลา 2-3 วันนั่นก็คือ

ไม่อ่านซิมและเครื่องรีสตาร์ทเอง

หลังจากที่เกิดอาการนี้ก็เลยเอาซิมการ์ดไปใส่เครื่องอื่น (Lumia 625) แล้วเครื่องอื่นก็อ่านซิมการ์ดได้ตามปกติ ก็แสดงว่าไม่น่าจะเป็นที่ซิมการ์ด

ด้านล่างนี้เป้นวิธีที่ส่งผลต่อข้อมูลของท่านเอง กรุณา backup ให้พร้อม

วิธีการทำ Factory Reset เรียงจากความเสี่ยงในการ brick ต่ำไปสูง (เหรอ)

บอกไว้ก่อนว่าเรื่องรูปภาพแต่ละขั้นตอน เราไม่ได้เก็บไว้ เพราะเครื่องรีสตาร์ทเองทุกครั้งที่เปิดเครื่องครับ

อย่างแรก ทำ Factory Reset ผ่านโปรแกรม Settings ของ ASUS เอง

อันนี้ทำได้ไม่ยากนัก

แนะนำให้เข้าที่แอป Settings -> Backup & Reset -> แล้วสั่ง Factory Reset ไปเลย

ขั้นตอนที่สอง ทำ Factory Reset แบบล้างเครื่องไปเลย

อันนี้เราปิดเครื่องก่อน แล้วค่อยกดปุ่ม Power + Volume Up นานซักระยะหนึ่ง เครื่องจะบูตขึ้นมาเป็นหน้าจอฟ้าๆ ที่มีเมนูตามรูปด้านบนสุด แล้วเราก็สั่ง Factory Reset ก็จะล้างเครื่องได้แล้ว

ขั้นตอนสุดท้าย ลงรอมใหม่เลยผ่านตัว recovery

ก่อนจะทำตามด้านล่างให้โหลดไฟล์ได้จากเว็บของ ASUS เองนะครับ เมื่อโหลดเสร็จก็ทำตามด้านล่างได้เลย

อันนี้ให้ทำตามขั้นตอนที่สอง แต่เปลี่ยนจาก Factory Reset เป็นเข้า Recovery แทน

เมื่อเข้าหน้า Recovery จะเห็นตัวหุ่นยนต์นอนแอ้งแม้งอยู่ กด Volume Down นานซักระยะหนึ่ง แล้วกด Volume Up ตาม จากนั้นปล่อย Volume Up แต่ยังกด Volume Down ค้างอยู่ก็จะขึ้นเมนูให้เลือก

เราเลือก Apply Updates from ADB (น่าจะสะกดอย่างนี้) แล้วเครื่องจะรอให้เราใส่คำสั่งลงไป (ตามรูปด้านบน) คำสั่งที่เราจะต้องพิมพ์คือ

adb sideload (ตำแหน่งไฟล์ zip ที่โหลดมา)

จากนั้นกด Enter ไฟล์ที่เราต้องการจะอัพโหลดเข้ามือถือแล้วโปรแกรมในมือถือจะจัดการให้เอง เมื่อทำเสร็จแล้วเลือก Reboot

ทั้งหมดที่กล่าวมา

เราทำมาหมดแล้วผลก็คือเครื่องไม่อ่านซิม และรีสตาร์ทเอง ทีนี้จะทำยังไงดีล่ะ

ตอนแรกค้นหาวิธีอย่างอื่นที่จะทำให้กลับมาใช้ได้ ผลก็ปรากฏว่าใช้ไม่ได้เหมือนเดิมอยู่ดี เราก็เลยตัดสินใจทำในสิ่งที่คนปกติจะทำกัน เพราะเครื่องยังมีประกันอยู่นั่นก็คือ

ส่งศูนย์ ASUS

แล้วศูนย์ ASUS ในกรุงเทพฯ อยู่ไหนล่ะ เรานึกว่าอยู่ที่เดิมตรงชั้นใต้ดิน Fortune Town ปรากฏว่าเข้าใจผิด

ศูนย์ย้ายไปนานแล้วเราก็ลองค้นหาใน Google ก็อยู่ที่อาคารชำนาญ เพ็ญชาติ บิสเนส เซ็นเตอร์ตรงแถวๆ พระราม 9 ใกล้กันกับตึก Fortune Town เดี๋ยววันนี้จะเอาเครื่องไปส่งศูนย์ ผลเป็นอย่างไรนี่ก็ไม่ทราบนะ หวังว่าจะใช้เวลาซ่อมไม่นานนัก

ตอนนี้กลายเป็นโทรศัพท์เครื่องแรกที่ส่งศูนย์เร็วที่สุด ฟังดูแล้วเป็นสถิติที่ไม่น่ายินดีนัก

Update (17/09/2557)

วันนี้โทรศัพท์หา call center ของบริษัท ASUS แล้วสอบถามสถานะการซ่อมของโทรศัพท์เครื่องนี้พบว่าอะไหล่เมนบอร์ดมาเรียบร้อยแล้วตั้งแต่เมื่อวานจะใช้เวลาประมาณ 3 วันทำการจึงจะเสร็จเรียบร้อย แต่เวลาผ่านไปประมาณ 1-2 ชั่วโมงพบว่ามี SMS จากทางศูนย์บอกว่าให้ไปรับโทรศัพท์ที่ศูนย์ได้เลย

ต้องการสื่อสารอย่างเร่งด่วนที่สุด...การโทรศัพท์ดีกว่าการส่งข้อความ

สมัยนี้คนมี Smartphone ที่ใช้ Android, iOS และ Windows Phone กันเยอะขึ้น ซึ่งรวมถึงเราที่ใช้มือถือ Android และ Windows Phone

นอกจากนี้คนยังใช้สื่อสังคมออนไลน์แนวๆ Facebook, Twitter, LINE ในการส่งข้อความไปให้บุคคลที่เราต้องการมากกว่าการส่ง SMS เนื่องจาก

มันฟรี

และยังเป็นวิธีการสื่อสารที่สะดวกอีกทางหนึ่ง

อย่างไรก็ตามเมื่อถึงเวลาที่เร่งด่วนคนมักจะส่งข้อความมาทางบุคคลที่ต้องการ ปกติแล้วถ้าเปิดการแจ้งเตือนไว้ตลอดเวลาก็คงไม่ใช่ปัญหาหรอกครับ

แต่บางคนไม่เป็นเช่นนั้นเพราะ...

บางคนอาจจะไม่รู้ว่ามีการแจ้งเตือน หรืออาจจะปิดการแจ้งเตือนเลยไม่ทราบข่าว หรือไม่ทราบข้อความที่คนนั้นส่งมาทำให้ลืมไปว่ามีเรื่องอะไรที่เร่งด่วนซึ่งรวมถึงเราด้วย ทำให้คนที่ส่งข้อความบางคนมารู้สึกไม่ดีนักเมื่อบุคคลที่เราต้องการไม่ได้อ่าน

เรื่องนี้เรารู้สึกว่าการส่งข้อความทางสื่อสังคมออนไลน์แนวนี้มักจะเป็นเรื่องที่ไม่เร่งด่วนมากกว่านะ เพราะบางคนก็ไม่ได้เปิดดูข้อความเหล่านี้ตลอด 24 ชั่วโมงอยู่แล้ว หรือบางคนก็ปิดการแจ้งเตือนไปเนื่องจากทำให้รบกวนสมาธิ

นอกจากนี้เวลาที่เร่งด่วนจริงๆ เขาไม่ใช้การส่งข้อความทางสื่อสังคมออนไลน์แบบนี้หรอกนะ เขาจะ...

โทรศัพท์มาหาคุณเอง

และยิ่งถ้าโทรศัพท์ไม่รับสาย เขาอาจจะโทรศัพท์มาอีกหลายสายเลยทีเดียว ขั้นตอนนี้แหละ ถ้าเราไม่รับสายนี่อาจจะมีปัญหากับตัวคุณเองได้ เรื่องนี้เราเคยโดนมาแล้วตอนช่วงวันที่มีรัฐประหาร (หรือปฏิวัติ?) แล้วมีโทรศัพท์มาหาเราหลายสายมากในขณะที่กำลังดูหนังอยู่ ซึ่งปกติเวลาดูหนังจำเป็นต้องปิดเครื่องใช่ไหมครับ เราก็ปิดโทรศัพท์ในช่วงนั้นพอดี หลังจากที่ดูหนังเสร็จเปิดโทรศัพท์ก็เกิดความรู้สึกว่า

ชิบหาย

สรุป ถ้าเร่งด่วนจริงๆ ให้โทรศัพท์ไปหาคนนั้นเถอะครับ :)

เช็ค Forward LINE ว่าเป็นจริงหรือเท็จง่ายๆ แค่เสิร์ช Google

ปัจจุบันทุกคนต่างมาใช้ LINE มากขึ้นเพราะมันมีอยู่ในโทรศัพท์เกือบทุกรุ่นที่เป็น Android, iOS, Windows Phone และอืนๆ ที่ลง LINE ได้ เราก็มี account LINE อยู่เหมือนกันแต่ขี้เกียจในรูปดิสเพลย์เพราะขี้เกียจเลือกรูปหน้าตัวเองมาใส่นะ 55

นอกเรื่องนานเข้าเรื่องได้แล้วครับ

ปกติทุกคนมักจะมีกรุ๊ปในไลน์กันใช่ไหมครับ?

ถ้าตอบว่าใช่

บ่อยครั้งที่ในกรุ๊ปไลน์มักจะมีคนไม่กี่คนที่อายุเยอะๆ พึ่งใช้อินเตอร์เน็ต หรืออายุน้อยๆ อย่างเรานี่แหละครับ และมักจะแชร์ฟอร์เวิร์ดเมลตั้งแต่เมื่อนานมาแล้วก่อนที่จะมีโปรแกรมอย่าง LINE เกิดขึ้นที่มักจะจริงบ้างและไม่จริงบ้าง (ส่วนใหญ่มักจะไม่จริงตามที่เราเคยอ่านผ่านๆ นะ) คิดว่าเจตนาของคนที่แชร์ข้อความแต่ละข้อความส่วนใหญ่จะหวังดีกันทั้งนั้นนะครับเพียงแต่อาจจะไม่รู้ว่าข้อความไหนจริง หรือข้อความไหนไม่จริง

แล้วจะมีวิธีการเช็คได้อย่างไรล่ะ? ว่าสิ่งที่เขาแชร์มานั้นจริงหรือไม่จริง ปกติเรามักจะทำอย่างนี้นะ

หมายเหตุ รูปแต่ละรูปแคปจากหน้าจอมือถือ บางรูปที่มีข้อความอาจจะมองไม่เห็นนะครับ นอกจากนี้รูปอาจจะมีขนาดใหญ่ทำให้เน็ตมือถือหมดก่อนได้นะ

เมื่อเราเปิดอินเตอร์เน็ตในมือถือแล้วเราจะเห็นการแจ้งเตือนในไลน์ใช่ป่ะ (ในรูปปิดการแจ้งเตือนไว้) เราก็กดเข้าไปจะเห็นข้อความที่แชร์ในกรุ๊ปไลน์ตามรูปด้านล่างนี้

เมื่อเราเห็นแล้ว เราก็ลองใส่คีย์เวิร์ดที่เกี่ยวข้องกับข้อความที่แชร์ไปเสิร์ชใน Google ดูก็จะขึ้นผลการค้นหาตามรูปด้านล่าง

จากนั้นเราก็จะเห็นเว็บแนวๆ hoax-slayer.com (หรือเว็บอื่นๆ ประมาณ Pantip ก็ได้ครับ) เว็บ hoax-slayer.com เป็นเว็บที่รวบรวมข้อความฟอร์เวิร์ดเมลซึ่งมักจะไม่จริงโพสไว้ในเว็บไว้ค้นหาข้อความที่เป็นเท็จครับ แล้วเราก็กดเข้าไป

พอกดเข้าไปแล้วลองอ่านดูจะมีรายละเอียดที่เกี่ยวข้องกับข้อความที่แชร์ๆ กันมาว่าที่จริงแล้วเป้นอย่างไร แล้วเราก็จะเช็คได้ว่าเรื่องนี้แชร์มานั้นจริงหรือไม่จริงนะครับ

เมื่อได้ข้อมูลว่าไม่จริงแล้วก็แนะนำเขากลับไปอย่างสุภาพนะครับ (ในรูปก็ไม่ได้สุภาพเท่าไรเล้ยยยยย - -")

หมายเหตุ อันนี้เป็นแค่ตัวอย่างนะ ไม่จำเป็นต้องเข้าเว็บ hoax-slayer.com ก็ได้ เพราะมีเว็บบอร์ดหรือเว็บอื่นๆ ที่สามารถให้ข้อความแนวเดียวกันกับเว็บที่บอกไว้ข้างบนครับ :)

Update มีเว็บไทยอีกเว็บหนึ่งที่เขียนแนวคล้ายๆ กับเว็บที่กล่าวไปแล้วก็คือเว็บ http://trueortao.wordpress.com แต่เนื้อหาไม่อัพเดตไปนานแล้วครับ

เข้าเว็บที่ถูกบล็อกได้ผ่านมือถือโดยใช้ Onavo Extend

ช่วงนี้หลายๆ คนก็เข้าเว็บที่คุ้นเคยกันผ่านมือถือหรือแท็บเล็ตซึ่งก็รู้อยู่นะว่าเว็บแนวไหน แล้วปรากฏว่าเจอหน้าตาที่บอกว่าเว็บนี้ไม่เหมาะสม หรือการถูกเข้าถึงเว็บนี้ถูกระงับโดยหน่อยงานแห่งหนึ่ง แปลโดยรวมว่าแมร่ง

โดนบล็อก...

ทีนี้จะทำยังไงดีล่ะ

ปกติวิธีที่จะเข้าเว็บที่โดนหน่วยงานเหล่านั้นบล็อกมีหลายวิธีนะ ได้แก่

  • เช่าใช้ VPN ต่างประเทศ หรือ
  • เช่า DigitalOcean มาเปิด VPN
  • และอื่นๆ

ซึ่งแต่ละวิธีก็ไม่ฟรีนะ แต่ในบทความนี้เราจะแนะนำโปรแกรมหนึ่งบนมือถือที่ใช้ได้ทั้ง Android และ iOS นั่นก็คือออออ...Onavo Extend

โปรแกรม Onavo Extend คือโปรแกรมที่เชื่อมต่อ VPN เพื่อที่จะลด Bandwidth จากการใช้เน็ต 3G หรือ 4G ซึ่งผลของการเชื่อมต่อ VPN นี่แหละทำให้เราสามารถเข้าเว็บที่ถูกบล็อกได้นั่นเอง ส่วนวิธีการที่จะทำให้เราใช้ VPN ได้ก่อนอื่นก็ดาวน์โหลดได้จาก App Store (iOS) หรือ Google Play (Android)

เมื่อดาวน์โหลดเสร็จแล้วก็ทำตามด้านล่างนี้ได้เลย (ในรูปจะเป็นขั้นตอนการใช้งานบน Android)

ตอนแรกก็เข้าไปที่แอป Onavo Extend

พอเข้าแอปแล้วจะขึ้นหน้าจอที่พูดถึงการทำงานของแอปนี้ให้กด Continue

เมื่อกดเสร็จแล้วจะขึ้นหน้า VPN Connect ที่จะกล่าวประมาณว่าให้เราเปิด VPN นั่นเอง (การเปิด VPN โดยใช้โปรแกรมนี้ต้องใช้เน็ตมือถือนะครับ) ให้กด Connect to VPN

ตอนนี้เราก็จะต่อ VPN ได้เรียบร้อยแล้วครับ

สำหรับคนที่ต้องการตั้งค่าเพิ่มเติมให้มาที่หน้าด้านรูปข้างบนกดที่สามจุดมุมบนขวาแล้วไปที่ Settings จากนั้นจะพบหน้า Settings แล้วปรับการตั้งค่าต่างๆ ได้ตามต้องการตั้งแต่คุณภาพรูปภาพ, cache เป็นต้น

ปัญหาที่ (น่าจะ) พบบ่อยคือต่อ VPN ไม่ได้

วิธีแก้ (หรือเปล่า) ให้เช็คว่าเราต่ออินเตอร์เน็ตด้วย 3G หรือ 4G หรือไม่ เพราะโปรแกรมนี้ต้องใช้เน็ตมือถือเพื่อที่จะต่อ VPN ได้ครับ

ย้ายไฟล์ และโฟลเดอร์จำนวนมากไป server เครื่องหนึ่งไปอีกเครื่องหนึ่ง

วันนี้ได้ทำภาระกิจที่ (โคตรจะ) ยิ่งใหญ่เลยก็คือย้ายข้อมูลจากเครื่อง server เครื่องเก่าไปยังเครื่องใหม่ ฟังดูแล้วน่าจะเป็นงานที่ไม่ยากซักเท่าไรนัก ก็แค่ย้ายไม่กี่ไฟล์ก็น่าจะเพียงพอแล้ว

ใช่ครับ งานอาจจะฟังดูไม่ยากนักกรณีที่ถ่ายโอนไฟล์แค่ไม่กี่ไฟล์ที่มีขนาดไม่ใหญ่มาก แต่ในกรณีนี้เราย้ายข้อมูลทั้งหมดประมาณ 2 ล้านกว่าๆ ไฟล์ แต่ละไฟล์ก็มีขนาดที่แตกต่างกันไปขึ้นกับคนที่อัพโหลด รวมๆ แล้วไฟล์ทุกไฟล์ใช้พื้นที่ไปประมาณ 200 - 300 GB ความรู้สึกตอนนั้นก็ประมาณว่า

ม่ายยยยยยยยยยยยยยยยยย

แล้วจะยังไงดีล่ะ เราลองทำหลายวิธีมาก่อนแล้ว ได้แก่

ทำผ่าน FTP

ทำได้โดยให้ก๊อปไฟล์ผ่านทาง FTP นี่แหละ ตอนนั้นเราใช้ mc (Midnight Commander) เข้าไปในโฟลเดอร์ที่ต้องการอัพโหลดโดยใช้

cd ftp://username@domainname.com - เปลี่ยน username และ domainame.com เป็นอย่างอื่นก็ได้ตามต้องการ

ทีนี้เมื่อเข้ามาแล้วเราก็จะก๊อปปี้ไฟล์ลงไปใน server อีกเครื่องหนึ่งทำได้โดยเลือดไฟล์หรือโฟลเดอร์ที่ต้องการแล้วกดปุ่ม F5 หลังจากนี้เราก็จะปล่อยให้โปรแกรมก๊อปไฟล์นะครับ แต่ในกรณีของเราที่กีอปไฟล์จำนวนมากทีนี้เวลาก๊อปไฟล์เป็นจำนวนมากอาจจะไม่สะดวก หรืออาจจะทำให้เห็นว่าโปรแกรมแม่งค้างไปแล้ววววนั่นแหละ จริงๆ ก็สามารถเช็คได้ว่าโปรแกรมทำอะไรอยู่ โดยเราจะใช้

strace -p PID

PID คือ Process ID ของโปรแกรมนั้นทำได้โดยเช็คใน ps [ชื่อโปรแกรม])

ทำผ่าน rsync

ทีนี้เราก็จะใช้อีกวิธีหนึ่งก็คือการก๊อปไฟล์จำนวนมากผ่านโปรแกรมที่ชื่อ rsync วิธีนี้ก็ทำได้โดย

rsync -av [path ของโฟลเดอร์ต้นทาง] [path ของโฟลเดอร์ปลายทาง]

ส่วน -av จะอธิบายในด้านล่างนี้นะครับ

  • a (achive) ให้ก๊อปไฟล์และ directories ทั้งหมด
  • v (verbose) ให้แสดงรายละเอียดของไฟล์และ directories

ข้อดีของโปรแกรมนี้นอกจากจะก๊อปไฟล์ได้ทีละเป็นจำนวนมากแล้ว ยังทำผ่าน ssh กับ backup ข้อมูลได้ด้วยนั่นเอง ซึ่งเราทำผ่าน ssh โดยพิมพ์ดังนี้

rsync -av [path ของโฟลเดอร์ต้นทาง] username@host:[path ของไฟล์ปลายทาง]

ทีนี้เราก็จะก๊อปปี้ไฟล์จำนวนมากผ่านทาง ssh ได้เรียบร้อยแล้ว แต่ในกรณีของเราก๊อปไฟล์เป็นจำนวนมาก และความเร็วของสายแลนที่ช้า (100 MBps) เลยต้องใช้เวลานานหลายชั่วโมงกว่าจะก๊อปปี้ให้เสร็จเรียบร้อย

วิธีเช็คว่าก๊อปเสร็จหรือยัง

ระหว่างที่ทำการก๊อปปี้ไฟล์อยู่ก็เช็คได้ว่าก๊อปเสร็จหรือยัง ของเราก็จะเช็คว่าไฟล์ระหว่างเครื่องต้นทางกับปลายทางมีจำนวนที่เท่ากันหรือเปล่าโดยใช้

 ls -1 [path ของโฟลเดอร์ที่เราต้องการ] | wc -l

เมื่อพิมพ์เสร็จแล้วกด Enter จะขึ้นจำนวนไฟล์ที่มีในโฟลเดอร์ตาม path นั้นเอง

สำหรับ Windows

จริงๆ โปรแกรม FTP กับ rsync ก็มีบน Windows เช่นกัน แต่ตัว FTP อาจจะไม่ต้องทำผ่าน Command Line แต่ทำผ่านโปรแกรมบน Windows เลยเช่น FileZilla, Cyberduck และอื่นๆ ครับ

หมายเหตุ

วิธีที่พูดถึงอันนี้พูดจากที่เคยทำนะครับ อาจจะไม่ถูกต้องซะทีเดียว ลองเช็คข้อมูลเพิ่มเติมจากเว็บอื่นอีกทีนะ
